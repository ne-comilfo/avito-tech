# Сервис объявлений

Данный проект — это веб-приложение для управления объявлениями, разработанное в рамках тестового задания для стажировки в Авито (весна 2026). Основная цель проекта — создание интуитивно понятного интерфейса для работы с товарами и интеграция с локальной языковой моделью для автоматизации генерации контента.

## О проекте
В рамках приложения реализованы следующие возможности:
* **Управление объявлениями**: Полный цикл просмотра и редактирования товаров в категориях «Транспорт», «Недвижимость» и «Электроника».
* **Интеграция с LLM**: Использование локальной нейросети (Ollama) для интеллектуальной оценки рыночной стоимости и создания продающих описаний.
* **Сохранность данных**: Механизм автосохранения черновиков в localStorage, предотвращающий потерю введенной информации при случайной перезагрузке страницы.
* **Навигация**: Оптимизированная обработка истории переходов для бесшовного пользовательского опыта.

## Самостоятельные решения

В ходе разработки были приняты архитектурные и функциональные решения, выходящие за рамки базового ТЗ, для улучшения стабильности, масштабируемости и UX приложения:

* **Доработка Backend:**
    * **PATCH-запросы:** Реализована логика обновления объявлений на стороне сервера.
    * **Сортировка:** Реализована обработка параметров сортировки по цене (`price-asc`/`price-desc`) на стороне сервера.
    * **Валидация:** Добавлена проверка типов данных, приходящих в PATCH-запросах, для предотвращения порчи базы данных некорректными значениями.
    * **Инфраструктура:** Оптимизированы скрипты запуска для одновременной работы фронтенда и бэкенда.
* **Улучшение UX и навигации:**
    * **Обработка ошибок:** Реализована кастомная страница **404 (Not Found)** с понятным интерфейсом и кнопкой возврата на главную.
    * **Навигация:** На страницу карточки товара добавлена кнопка «Назад», упрощающая пользовательский путь (User Flow).
* **Бизнес-логика редактирования:** В форме редактирования заблокирована возможность смены категории. Это осознанное решение, так как изменение категории влечет за собой смену набора технических характеристик, что в рамках текущей БД может привести к конфликтам данных.
* **Интеллектуальный помощник:** Реализован механизм автоматического извлечения данных из ответов ИИ. При нажатии кнопки «Применить» система парсит текст, находит предложенную стоимость и автоматически подставляет её в соответствующее поле ввода.

## Инструкция по запуску

### 1. Подготовка ИИ (Ollama)
Проект использует модель **Gemma 2 (2b)** через интерфейс Ollama.

1. Установите Ollama с [официального сайта](https://ollama.com).
2. Загрузите модель через терминал:
   ```bash
   ollama pull gemma2:2b
Запустите сервер Ollama с поддержкой CORS:

Windows (PowerShell):
```bash 
$env:OLLAMA_ORIGINS="*"; ollama serve
```

macOS / Linux:
```bash 
 OLLAMA_ORIGINS="*" ollama serve
```

### 2. Запуск приложения
Сначала склонируйте репозиторий:

``` bash
git clone "https://github.com/ne-comilfo/avito-tech.git"
```

## Запуск сервера (Backend):

```bash
cd server
npm install
npm start
```
## Запуск клиента (Frontend):
Откройте новое окно терминала:

```bash
cd client
npm install
npm run dev
```

## Контроль качества кода
Проект прошел проверку на соответствие стандартам разработки (использовать только в папке client):

```bash 
npm run lint
```
— проверка отсутствия ошибок линтера (ESLint).

```bash 
npm run format 
```
— проверка корректности форматирования (Prettier).

Примечание для проверки
Если функции AI (оценка цены или генерация описания) выдают ошибку, убедитесь, что локальный сервер Ollama запущен с переменной окружения OLLAMA_ORIGINS="*"
